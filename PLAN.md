# План работ: функции кнопок Referent

Основание: [PROJECT.md](./PROJECT.md).

Цель: по URL статьи парсить содержимое и по кнопке подключать AI для нужного действия. Результат — текст, процент плагиата или изображение.

---

## Текущее состояние

| Кнопка | Статус | Что есть сейчас |
|--------|--------|-----------------|
| О чем статья | Частично | Парсинг → JSON в интерфейсе, **без AI-описания** |
| Тезисы | Частично | Парсинг → JSON, **без AI-тезисов** |
| Пост для Telegram | Частично | Парсинг → JSON, **без AI-поста** |
| Перевод | Готово | `/api/translate`, `lib/translate-article.ts`, OpenRouter |
| Изображение | Готово | `/api/image`, `lib/generate-image.ts`, Grok через OpenRouter |
| Плагиат | Готово | `/api/plagiarism`, `lib/plagiarism.ts`, rerank + локальный fallback |

Общая инфраструктура уже есть: парсинг (`lib/parse-article.ts`), клиент OpenRouter (`lib/openrouter.ts`), UI (`app/components/article-analyzer.tsx`), health-check (`/api/health`).

## Порядок действий

### Этап 0. Подготовка

1. Сверить `PROJECT.md` с фактическим UI: в приложении **6 кнопок** (добавлен «Перевод»); при необходимости обновить PROJECT.md и README.md.
2. Зафиксировать единый поток: **URL → парсинг → AI-действие → форматированный результат в UI**.
3. Выбрать текстовую модель OpenRouter для summary/theses/telegram (например, `poolside/laguna-xs-2.1` или отдельная переменная `OPENROUTER_TEXT_MODEL`).
4. Добавить переменную в `.env.example` и документировать для Vercel.

---

### Этап 1. Общая библиотека для текстовых AI-действий

1. Создать `lib/ai-text.ts` (или три отдельных модуля) с общей логикой:
   - получение клиента OpenRouter;
   - обрезка `content` до лимита токенов (как в `translate-article.ts`, ~12 000 символов);
   - единая обработка ошибок и пустых ответов модели.
2. Для каждого действия — свой system-prompt и формат ответа:
   - **О чем статья** → короткий текст (2–4 предложения), не JSON;
   - **Тезисы** → маркированный список (5–10 пунктов);
   - **Пост для Telegram** → готовый пост: заголовок, абзацы, эмодзи по желанию, до ~4096 символов.
3. Экспортировать функции: `summarizeArticle`, `generateTheses`, `generateTelegramPost`.

---

### Этап 2. API-маршруты

Для каждой из трёх недостающих кнопок — отдельный route по образцу `/api/translate`:

| Маршрут | Вход | Выход |
|---------|------|--------|
| `POST /api/summary` | `{ url }` | `{ text: string }` |
| `POST /api/theses` | `{ url }` | `{ items: string[] }` или `{ text: string }` |
| `POST /api/telegram` | `{ url }` | `{ text: string }` |

Порядок внутри каждого route:

1. Валидация URL.
2. `parseArticle(url)` из `lib/parse-article.ts`.
3. Вызов соответствующей AI-функции.
4. Ответ JSON + понятные HTTP-ошибки (400/502).

Уже готовые маршруты **не трогать**, только при необходимости вынести общий helper `parseUrlFromBody`.

### Этап 3. Кнопка «О чем статья»

1. Реализовать `lib/summarize-article.ts` + `POST /api/summary`.
2. В `article-analyzer.tsx`:
   - для `action === "summary"` вызывать `/api/summary`, а не показывать сырой JSON;
   - добавить тип результата `{ type: "text"; text: string }`.
3. В блоке «Результат» — обычный текст с `whitespace-pre-wrap`, не `<pre>` с JSON.
4. Подпись загрузки: «Генерация описания…».

---

### Этап 4. Кнопка «Тезисы»

1. Реализовать `lib/generate-theses.ts` + `POST /api/theses`.
2. Промпт: выделить главные мысли, без воды, на русском (или на языке статьи — зафиксировать в промпте).
3. UI: список `<ul><li>` или нумерованный список; тип `{ type: "theses"; items: string[] }`.
4. Подпись загрузки: «Формирование тезисов…».

---

### Этап 5. Кнопка «Пост для Telegram»

1. Реализовать `lib/generate-telegram-post.ts` + `POST /api/telegram`.
2. Промпт: лаконичный пост, хук в первой строке, ссылка на источник (передавать `url` в промпт).
3. UI: текстовый блок + кнопка «Копировать» (опционально, но полезно).
4. Подпись загрузки: «Создание поста…».

## Критерии готовности

- Каждая кнопка вызывает **свой API** и показывает **понятный результат**, а не сырой JSON парсера.
- Текстовые действия используют OpenRouter с ключом из `OPENROUTER_API_KEY`.
- Ошибки (нет ключа, таймаут, блокировка сайта) отображаются в UI красным текстом.
- Локально (`npm run dev`) и на Vercel поведение совпадает при одинаковых env-переменных.
