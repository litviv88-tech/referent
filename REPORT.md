## Цель

Подключить AI через OpenRouter для трёх кнопок, которые раньше показывали только сырой JSON после парсинга: **«О чем статья»**, **«Тезисы»**, **«Пост для Telegram»**.

---

## Что было сделано

### Этап 1. Общая библиотека для текстовых AI-действий

| Файл | Назначение |
|------|------------|
| `lib/ai-text.ts` | Общая логика: модель `OPENROUTER_TEXT_MODEL`, обрезка текста до 12 000 символов, `chatCompletion`, парсинг JSON из ответа |
| `lib/summarize-article.ts` | `summarizeArticle()` — краткое описание статьи (2–4 предложения) |
| `lib/generate-theses.ts` | `generateTheses()` — 5–10 тезисов в виде массива |
| `lib/generate-telegram-post.ts` | `generateTelegramPost()` — готовый пост с ссылкой на источник |
| `lib/api-url.ts` | Валидация URL из тела запроса (общий helper для API) |

Модель по умолчанию: `poolside/laguna-xs-2.1` (переменная `OPENROUTER_TEXT_MODEL`).

---

### Этап 2. API-маршруты

| Маршрут | Метод | Вход | Выход |
|---------|-------|------|-------|
| `/api/summary` | POST | `{ url }` | `{ text: string }` |
| `/api/theses` | POST | `{ url }` | `{ items: string[] }` |
| `/api/telegram` | POST | `{ url }` | `{ text: string }` |

Каждый маршрут: валидация URL → `parseArticle(url)` → вызов AI-функции → JSON-ответ или ошибка 400/500.

---

### Этап 3. Кнопка «О чем статья»

- Клиент вызывает `/api/summary` вместо показа JSON парсера.
- Результат отображается как обычный текст (`whitespace-pre-wrap`).
- Подпись загрузки: **«Генерация описания…»**.

---

### Этап 4. Кнопка «Тезисы»

- Клиент вызывает `/api/theses`.
- Результат — нумерованный список `<ol>`.
- Подпись загрузки: **«Формирование тезисов…»**.

---

### Этап 5. Кнопка «Пост для Telegram»

- Клиент вызывает `/api/telegram`.
- В промпт передаётся URL источника для ссылки в конце поста.
- Результат — текстовый блок + кнопка **«Копировать»**.
- Подпись загрузки: **«Создание поста…»**.
